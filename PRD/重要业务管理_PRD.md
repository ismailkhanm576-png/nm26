# 重要业务管理 需求文档 (PRD)

## 一：需求背景
### 1、需求背景
随着政企业务的不断发展，部分核心客户和关键业务（如党政军、金融、大企业客户）对网络质量和故障恢复时间有着极高的要求。为了更好地对这些高价值、高保障要求的业务进行针对性监控、主动运维和资源倾斜，系统需要提供“重要业务管理”功能。通过建立重要业务台账，实现对关键专线的标识、纳管和等级划分，从而提升高价值客户的网络保障能力和满意度。

### 2、业务场景
- **业务纳管**：运维人员需要从全量业务库中，将特定的高保障专线检索出来，并加入到“重要业务库”中进行特殊关注。
- **等级划分**：运维人员需要对已纳管的重要业务进行等级评定，划分为“重要”或“特别重要”，以便在故障发生时决定处理优先级。
- **日常维护**：运维人员需要定期维护重要业务清单，包括批量导入新增业务、导出台账报表、修改现有业务的重要等级，以及将不再需要重点保障的业务移出该清单。
- **快速检索**：运维人员需要通过多维度条件（如客户名称、业务类型、保障等级、地市等）快速定位特定的重要业务。

### 3、业务对象
- **重要业务记录（Important Business Record）**：
  - **客户信息**：客户名称、客户编号。
  - **业务信息**：业务类型（数据专线、互联网专线、MPLS-VPN专线、语音专线）、产品实例。
  - **保障信息**：重要等级（重要/特别重要）、A端/Z端保障等级（AAA/AA/A/普通）。
  - **位置信息**：A端/Z端地市、区县、详细地址。

## 二：需求概述
“重要业务管理”模块主要提供对政企重要专线业务的台账管理功能。核心功能包括：重要业务的多条件综合查询、业务数据的批量导入与导出、从现有业务库中手动查询并添加重要业务、重要等级的设置与修改，以及重要业务的单条/批量删除。通过该模块，运维人员能够建立并动态维护一份准确的重要业务清单，为后续的故障优先处理、主动巡检等高阶运维场景提供基础数据支撑。

## 三：需求描述
### 1、视图关系（菜单路径）
- **菜单路径**：专线(新) -> 重要业务管理

### 2、流程说明
- **添加重要业务流程**：
  1. 用户点击底部“添加”按钮。
  2. 系统弹出“重要业务查询添加”弹窗。
  3. 用户输入查询条件检索全量专线业务，勾选需要纳管的业务后点击“确定”。
  4. 系统弹出“重要等级选择”弹窗。
  5. 用户选择“重要”或“特别重要”等级，点击“确定”。
  6. 系统将选中的业务加入重要业务列表，并赋予对应的重要等级。
- **修改重要等级流程**：
  1. 用户在列表中找到目标业务，点击操作列的“修改”图标。
  2. 系统弹出修改弹窗，用户重新选择等级（重要/特别重要）。
  3. 点击“确定”保存修改，列表数据更新。
- **删除流程**：
  1. 用户勾选单条或多条业务（或直接点击单条操作列的删除图标），点击“删除”按钮。
  2. 系统弹出二次确认弹窗。
  3. 用户点击“确定删除”，数据从重要业务列表中移除。
- **导入导出流程**：
  - **导入**：点击“导入模版”下载标准Excel模版 -> 按照模版填写数据 -> 点击“导入”上传文件 -> 系统解析并更新列表。
  - **导出**：点击“导出”按钮，系统将当前查询条件下的重要业务数据导出为Excel文件。

### 3、界面原型
界面整体采用上下布局，分为三个主要区域：
1. **顶部查询条件区**：包含关键字、重要等级、业务类型等多个筛选条件及查询/置回按钮。
2. **中间数据列表区**：展示重要业务的详细字段信息，支持横向/纵向滚动，包含单行操作按钮。
3. **底部操作与分页区**：包含导入、导出、添加、批量删除等全局操作按钮，以及已选数据统计和分页组件。

### 4、功能说明

#### 4.1 查询条件区
- **4.1.1 关键字查询**
  - **控件类型**：文本输入框。
  - **功能说明**：支持输入“客户名称”、“客户编号”或“业务地址”（A端或Z端地址）进行模糊匹配查询。
- **4.1.2 重要等级筛选**
  - **控件类型**：下拉选择框。
  - **选项内容**：全部（默认）、重要、特别重要。
  - **功能说明**：根据业务在系统内被标记的重要等级进行精确筛选。
- **4.1.3 业务类型筛选**
  - **控件类型**：下拉选择框。
  - **选项内容**：业务类型（默认/全部）、数据专线、互联网专线、MPLS-VPN专线、语音专线。
  - **功能说明**：根据专线的具体业务类型进行精确筛选。
- **4.1.4 产品实例查询**
  - **控件类型**：文本输入框。
  - **功能说明**：支持输入“产品实例”标识进行模糊匹配查询。
- **4.1.5 保障等级筛选**
  - **控件类型**：下拉选择框。
  - **选项内容**：保障等级（默认/全部）、AAA、AA、A、普通。
  - **功能说明**：匹配A端或Z端的保障等级进行筛选（满足任一端即可）。
- **4.1.6 地市筛选**
  - **控件类型**：下拉选择框。
  - **选项内容**：地市（默认/全部）、内蒙古自治区各个地市（如呼和浩特、包头等）。
  - **功能说明**：匹配A端或Z端所属地市进行筛选（满足任一端即可）。
- **4.1.7 查询与置回操作**
  - **查询按钮**：点击后，根据上述所有已填写的条件进行组合查询（AND逻辑），刷新下方列表数据，并将分页重置为第一页，清空已勾选的记录。
  - **置回按钮**：点击后，清空所有查询条件（恢复默认空值），刷新列表显示全部数据，重置分页并清空勾选状态。

#### 4.2 数据列表区
- **4.2.1 列表字段展示**
  - **展示字段**：复选框、重要等级、客户名称、客户编号、业务类型、产品实例、A端保障等级、A端地市、A端区县、A端地址、Z端保障等级、Z端地市、Z端区县、Z端地址、操作。
  - **样式规范**：
    - 表头固定，列表支持垂直和水平滚动。
    - “重要等级”和“保障等级”采用彩色徽标（Badge）展示（如：特别重要/AAA为红色，重要/AA为橙色，A为黄色，普通为蓝色）。
    - 数据为空时，表格内展示提示语“暂无匹配的重要业务数据”。
- **4.2.2 列表勾选功能**
  - **功能说明**：表头提供全选复选框（仅全选当前页数据），每行首列提供单选复选框。勾选后当前行背景高亮，用于支持底部的批量删除操作。
- **4.2.3 单行操作 - 修改重要等级**
  - **控件类型**：编辑图标按钮。
  - **功能说明**：点击弹出“修改重要等级”弹窗，可对当前行的业务等级进行修改。
- **4.2.4 单行操作 - 删除**
  - **控件类型**：垃圾桶图标按钮。
  - **功能说明**：点击弹出删除确认弹窗，确认后将该条业务从重要业务库中移除。

#### 4.3 底部操作与分页区
- **4.3.1 导入与导出操作**
  - **导入模版**：点击下载重要业务批量导入的标准Excel模版文件。
  - **导入**：点击调起本地文件选择器，限制选择 `.xlsx, .xls` 格式文件。选择文件后执行导入逻辑，将数据追加至列表。
  - **导出**：点击将当前查询条件过滤后的重要业务数据导出为Excel文件。
- **4.3.2 添加重要业务**
  - **控件类型**：主要按钮（Primary Button）。
  - **功能说明**：点击弹出“重要业务查询添加”弹窗，触发添加流程。
- **4.3.3 批量删除**
  - **控件类型**：危险操作按钮（Danger Button）。
  - **功能说明**：当列表有勾选记录时激活，未勾选时置灰禁用。点击弹出二次确认弹窗，确认后批量移除选中的业务。
- **4.3.4 分页与统计**
  - **已选统计**：实时显示当前已勾选的记录数量。
  - **分页组件**：展示总记录数，支持切换每页显示条数（PageSize），支持上一页、下一页及指定页码跳转。切换分页或条数时，自动清空当前的勾选状态。

#### 4.4 弹窗模块
- **4.4.1 重要业务查询添加弹窗**
  - **触发机制**：点击底部“添加”按钮。
  - **功能说明**：复用系统的通用业务查询组件，支持通过关键字、地市等条件检索全量专线业务（系统底层自动过滤掉“企宽”等非专线业务）。支持多选，选定后点击“确定”进入重要等级选择步骤。
- **4.4.2 重要等级选择/修改弹窗**
  - **触发机制**：添加业务的第二步，或点击列表单行的“修改”图标。
  - **功能说明**：提供单选组件（Radio），选项为“重要”和“特别重要”。包含“取消”和“确定”按钮。确认后更新对应业务的重要等级属性。
- **4.4.3 删除确认弹窗**
  - **触发机制**：点击单行删除或批量删除按钮。
  - **功能说明**：展示警告信息“确定要删除选中的 X 条重要业务数据吗？此操作不可恢复。”。包含“取消”和“确定删除”按钮，点击确定后执行删除逻辑并关闭弹窗。
