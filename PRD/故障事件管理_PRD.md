# 故障事件管理 需求文档 (PRD)

## 一：需求背景
### 1、需求背景
随着专线业务规模的扩大和网络复杂度的提升，及时发现、准确定位并快速响应网络故障成为保障政企客户网络质量的关键。为了提升故障处理效率，实现故障事件的集中监控、自动化识别和及时通知，系统需要提供“故障事件管理”功能。通过该功能，运维人员可以实时监控故障事件状态，灵活配置故障识别规则，并精准设置故障短信通知策略，从而构建闭环的故障主动运维体系。

### 2、业务场景
- **故障实时监控**：值班人员需要实时查看当前网络中发生的故障事件，了解受影响的客户、业务类型及故障识别结果（如业务中断或保护降级），并能查看故障发生时的网络快照。
- **故障规则配置**：系统管理员需要根据网络设备的告警特征（告警对象、资源界面、告警标题等），自定义配置故障识别规则，以便系统底层引擎能准确将底层告警映射为上层业务故障（中断或降级）。
- **故障通知配置**：客户经理或运维主管需要针对特定的高价值产品实例，配置故障短信接收人（姓名、电话），确保在故障发生时，相关责任人能第一时间收到短信告警。

### 3、业务对象
- **故障事件（Fault Event）**：包含事件状态、事件编号、客户信息、业务类型、产品实例、电路代号、识别结果、事件/故障/恢复时间及快照。
- **故障识别规则（Fault Rule）**：包含告警对象、传输资源界面、告警标题、识别结果及最后修改时间。
- **短信发送配置（SMS Config）**：包含产品实例、短信受理人（姓名、电话）、业务类型及客户信息。

## 二：需求概述
“故障事件管理”模块主要提供故障的集中监控与策略配置功能。该模块包含三个子功能：
1. **故障事件监控**：提供多维度的故障事件查询、列表展示、导出及故障快照查看功能。
2. **故障识别规则**：提供故障映射规则的增删改查功能，支持按告警对象、资源界面等条件配置业务中断或保护降级的识别逻辑。
3. **故障短信发送配置**：提供基于产品实例的短信接收人配置管理，支持多受理人的动态添加与批量管理，确保故障信息精准触达。

## 三：需求描述
### 1、视图关系（菜单路径）
- **菜单路径**：故障事件管理
  - 故障监控 -> 故障事件监控
  - 配置管理 -> 故障识别规则
  - 配置管理 -> 故障短信发送配置

### 2、流程说明
- **故障监控流程**：用户进入“故障事件监控”页面 -> 设置查询条件（如状态、时间段） -> 查看故障列表 -> 点击“查看快照”分析故障详情 -> 导出故障数据报表。
- **规则配置流程**：用户进入“故障识别规则”页面 -> 点击“新增规则” -> 选择告警对象和资源界面，输入告警标题，勾选识别结果 -> 确认保存 -> 系统根据新规则进行后续的故障识别。
- **短信配置流程**：用户进入“故障短信发送配置”页面 -> 点击“新增配置” -> 通过业务查询弹窗选择目标产品实例 -> 动态添加或移除短信受理人（姓名、电话） -> 确认保存。

### 3、界面原型
界面整体采用左侧导航树+右侧内容区的布局。
- **左侧导航**：包含“故障监控”和“配置管理”两个分组，支持折叠/展开。
- **右侧内容区**：顶部为页签栏（支持多开和右键关闭），下方为具体的业务视图。每个业务视图均采用“顶部查询条件区 + 中间数据列表区 + 底部操作与分页区”的标准上下布局。

### 4、功能说明

#### 4.1 故障事件监控
- **4.1.1 查询条件区**
  - **关键字查询**：文本输入框，支持按“客户名称/客户编号/产品实例/电路代号”模糊查询。
  - **业务类型筛选**：下拉选择框，包含全部、数据专线、互联网专线、语音专线、MPLS-VPN专线、APN专线。
  - **识别结果筛选**：下拉选择框，包含全部、业务中断、保护降级。
  - **事件状态筛选**：下拉选择框，包含全部、活动、历史。
  - **产生时间筛选**：两个日期选择器，支持按开始日期和结束日期进行区间过滤。
  - **操作按钮**：提供“查询”（执行过滤并重置分页）和“重置”（清空条件并刷新）按钮。
- **4.1.2 数据列表区**
  - **列表展示**：展示事件状态（彩色徽标）、事件编号、客户编号、客户名称、业务类型、产品实例标识、电路代号、识别结果（彩色徽标）、事件产生时间、故障产生时间、故障恢复时间、操作（事件快照）。
  - **事件快照查看**：点击操作列的“查看快照”，弹出快照查看弹窗。
- **4.1.3 底部操作与分页区**
  - **导出**：点击“导出”按钮，将当前过滤后的故障事件数据导出。
  - **分页组件**：支持切换每页条数、页码跳转及总数展示。
- **4.1.4 故障快照查看弹窗**
  - **功能说明**：展示故障发生时的网络拓扑或状态截图。
  - **图片控制**：顶部工具栏提供“放大”、“缩小”、“重置”按钮，支持对图片进行缩放查看。底部居中悬浮显示当前缩放比例（如 100%）。

#### 4.2 故障识别规则
- **4.2.1 查询条件区**
  - **告警对象筛选**：下拉选择框，包含全部、网元、端口、单板。
  - **传输资源界面筛选**：下拉选择框，包含全部、端、路径。
  - **识别结果筛选**：下拉选择框，包含全部、业务中断、保护降级。
  - **告警标题查询**：文本输入框，支持模糊匹配。
  - **操作按钮**：查询、重置。
- **4.2.2 数据列表区**
  - **列表展示**：包含复选框（支持全选/单选）、告警对象、传输资源界面、告警标题、识别结果、最后修改时间、操作（修改、删除）。
  - **单行操作**：点击“修改”打开编辑弹窗；点击“删除”打开二次确认弹窗。
- **4.2.3 底部操作与分页区**
  - **全局操作**：提供“导出”、“新增规则”、“删除”（批量删除，勾选后激活）按钮。
  - **分页组件**：标准分页控件。
- **4.2.4 新增/修改规则弹窗**
  - **表单字段**：
    - 告警对象（必填）：下拉选择（网元/端口/单板）。
    - 传输资源界面（必填）：下拉选择（端/路径）。联动逻辑：若选择“端”，则识别结果不能包含“保护降级”。
    - 告警标题（必填）：多行文本框，支持输入多个标题（逗号分隔）。
    - 识别结果（必填）：复选框组（业务中断、保护降级）。
  - **保存逻辑**：校验必填项。若是新增且存在相同“告警对象+资源界面+识别结果”的规则，则将告警标题进行合并更新；否则创建新规则。
- **4.2.5 删除确认弹窗**
  - **功能说明**：单条或批量删除时弹出，提示“确定要删除选中的 X 条故障识别规则吗？”，确认后执行删除。

#### 4.3 故障短信发送配置
- **4.3.1 查询条件区**
  - **关键字查询**：文本输入框，支持按“客户名称/客户编号/产品实例”模糊查询。
  - **业务类型筛选**：下拉选择框，同上。
  - **操作按钮**：查询、重置。
- **4.3.2 数据列表区**
  - **列表展示**：包含复选框、产品实例、短信受理人（格式：姓名(电话)，多个逗号分隔）、业务类型、客户名称、客户编号、操作（编辑、删除）。
- **4.3.3 底部操作与分页区**
  - **全局操作**：导出、新增配置、删除（批量）。
  - **分页组件**：标准分页控件。
- **4.3.4 新增/编辑配置弹窗**
  - **产品实例选择**：点击只读文本框，弹出“业务查询选择”通用弹窗（复用 `ServiceSelectionModal`），支持多选，选定后回填实例标识。
  - **受理人管理**：
    - **动态列表**：支持添加多个受理人，每个受理人包含“姓名”和“电话”输入框。
    - **单条移除**：每行后方提供垃圾桶图标，仅剩一条时禁用。
    - **批量移除**：提供全选/单选复选框，勾选后点击右上角“删除(X)”按钮，弹出二次确认弹窗，确认后批量移除选中的受理人。
    - **添加受理人**：点击“添加受理人”按钮，在列表末尾新增一行空数据。
  - **保存逻辑**：校验产品实例和所有受理人信息是否填写完整，确认后保存。
- **4.3.5 移除受理人确认弹窗**
  - **功能说明**：在编辑配置弹窗中批量移除受理人时触发，提示“确定要移除选中的 X 位受理人吗？”，确认后从表单中移除。
- **4.3.6 删除配置确认弹窗**
  - **功能说明**：在主列表进行单条或批量删除时触发，确认后删除整条短信配置记录。
